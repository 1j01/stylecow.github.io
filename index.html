Why another CSS preprocessor?

The main purpose of stylecow is to bring more CSS support to all browsers. Stylecow converts your code to be more compatible with all browsers adding vendor prefixes, fixing issues, generating polyfills, etc.

You can play also with latests (or futures) css features such variables, nested rules or color manipulation using the w3c syntax. This mean you don't have to learn another language such LESS or SASS to use variables or color manipulation, you write css and you get css, and if you stop using stylecow, your css code remains css code.

Quick start

Use npm to install the library:

npm install stylecow --save

Stylecow by itself is not very useful, you have to install any plugin, for example color and variables:

npm install stylecow-plugin-color stylecow-plugin-variables --save

Ok, let's create the js file:

var stylecow = require('stylecow');

stylecow.setConfig({

	//Minimal browser supported
	support: {
		explorer: 8,
		firefox: 30,
		chrome: 35,
		safari: 5,
		opera: 22,
		android: 4,
		ios: 6
	},

	//Plugins to execute (without "stylecow-plugin-")
	plugins: [
		"color",
		"variables"
	],

	//Code style used (normal or minify)
	code: "minify"
});

//Load and convert a css file
css = stylecow.convertFromFile("styles.css");

//Get the minified code
var code = css.toCode();

Command line interface

To execute stylecow in the command line interface, you need install the stylecow-cli module. Use -g to install as global:

npm install stylecow-cli -g

Now you have the stylecow command. The next thing is a json file to store the configuration. To create it in an interactive way:

stylecow init

An typical config file looks like:

{
	"input": "styles.css",
	"output": "styles.min.css",
	"support": {
		"explorer": 8,
		"firefox": 30,
		"chrome": 35,
		"safari": 5,
		"opera": 22,
		"android": 4,
		"ios": 6
	},
	"plugins": [
		"color",
		"variables"
	],
	"code": "minify"
}

The input and output properties are optional you can define it later using --input and --output options

To execute the config file:

stylecow execute stylecow_config.json

You can use the option --watch to keep stylecow watching the css files and regenerate the output code when any input file changes.

Use stylecow execute -h to view all available options.

Using with Grunt

If you use Grunt as your task runner there is a plugin available that you can install with npm:

npm install grunt-stylecow --save-dev

Now, you can register and configurate it in the Grunt file:

module.exports = function(grunt) {
	grunt.initConfig({
	    stylecow: {
	        options: {
	            "support": {
	                "explorer": 10,
	                "firefox": 30,
	                "chrome": 35,
	                "safari": 6,
	                "opera": 22,
	                "android": 4,
	                "ios": 6
	            },
	            "plugins": [
	                "color",
	                "variables"
	            ],
	            "code": "normal"
	        },
	        dist: {
	            files: {
	                'css/styles.min.css': ['./css/styles.css']
	            }
	        }
	    }
	});

	grunt.loadNpmTasks('grunt-stylecow');

	grunt.registerTask('default', ['stylecow']);
}

Plugins

All stylecow operations are done via plugins. There are several plugins available, for example, to play with variables using the w3c syntax, to manipulate the colors using the CSS Colors Module Level 4 syntax, to play with the latest syntax of flexbox generating the old syntax for ie10 and webkit browsers, etc...

The available plugins are the following:

Support for CSS incoming features:

  • color
  • matches
  • nested-rules
  • variables

Utilities:

  • import (To insert the code of the @import rules with relative urls)
  • source-comment

Code fixing:

  • fixes
  • flex
  • initial
  • linear-gradient
  • prefixes
  • rem

Emulation of some features in old Internet Explorer using ms-filters:

  • msfilter-background-alpha
  • msfilter-linear-gradient
  • msfilter-transform

Why another CSS processor?

The main purpose of stylecow is to support modern CSS in all browsers. Stylecow transforms your code to be compatible with all browsers adding vendor prefixes, fixing issues, generating polyfills, etc.

You can use latest (or future) css features such variables, custom media, color manipulation, etc using the w3c syntax. So, you don't have to learn a different syntax as with preprocessors such LESS or SASS to use these features. Stylecow is a postprocessor, meaning it works with pure css. You write css and you get css, and if you stop using stylecow, your css code remains css code.

Why is stylecow special?

There are other css processors available such PostCSS or Rework. But Stylecow has the most powerful CSS parser and API among many other benefits:

  • Easy to install and use

    All available plugins are installed with this package, so it can be ready to use in few seconds.

  • Easy to extend and customize

    Stylecow provides a clean and good API to work with css and create your own plugins in a easy, safe and fast way.

  • Good syntax checking

    The Stylecow parser can detect any css syntax error and warn you in what line and column occurred.

  • Custom style code generation

    There are two styles by default (normal and minify) but more styles can be created easily. Source maps can also be created.

  • Development tools

    A watcher and a browser live-reload utility are included, so dont't spend your time setting, but developing.

  • Easily combinable

    There are plugins for gulp and grunt, so you can combine stylecow with other tools, even with other pre/postprocessors

Quick start

Use npm to install the library. Add -g if you want to install it globally:

npm install stylecow -g

Now you have the stylecow command with some useful utilities. The next step is to create a file with the configuration. You can create it in an interactive way using the command:

stylecow init

In this process, you have to define the following settings:

  • support: Minimum version supported by the browsers explorer, firefox, chrome, safari, opera, android and ios
  • plugins: The plugins you want to use
  • code: Code style used to output the css. (normal and minify)
  • input: path of the main css file to convert
  • output: path of the file where save the result
  • map: whether or not generate the source map

A typical config file looks like:

{
    "files": [
        {
            "input": "styles.css",
            "output": "styles.min.css",
            "map": "styles.min.map"
        }
    ],
    "support": {
        "explorer": 8,
        "firefox": 30,
        "chrome": 35,
        "safari": 5,
        "opera": 22,
        "android": 4,
        "ios": 6
    },
    "plugins": [
        "color",
        "variables",
        "nested-rules"
    ],
    "code": "minify"
}

By default the configuration file is saved as stylecow.json but you can choose a different name. To run this configuration, just execute:

stylecow

You can use the following options:

  • -c, --config <file> Change the default config file (./stylecow.json)
  • -w, --watch To keep watching the changes in the css files and generate the output automatically
  • -r, --live-reload Live reload the css in the browser when the output changes (using a websocket)

Using with Grunt/Gulp

Are you one of these guys? No problem, there is a grunt plugin available that you can install with npm:

npm install grunt-stylecow --save-dev

Now, you can register and configure it in the Grunt file:

module.exports = function(grunt) {
    grunt.initConfig({
        stylecow: {
            options: require('./stylecow.json'), //Your config file
            dist: {
                files: {
                    'css/styles.min.css': ['./css/styles.css']
                }
            }
        }
    });

    grunt.loadNpmTasks('grunt-stylecow');

    grunt.registerTask('default', ['stylecow']);
}

Or if you are a Gulp-guy, here is your plugin: 

var gulp = require('gulp');
var stylecow = require('gulp-stylecow');

gulp.task('stylecow', function() {
  gulp.src('./css/styles.css')
    .pipe(stylecow(require('./stylecow.json')))
    .pipe(gulp.dest('./dist/'))
});

Plugins

All stylecow operations are done via plugins. There are several plugins available, for example, to play with variables using the w3c syntax, to manipulate the colors using the CSS Colors Module Level 4 syntax, to play with the latest syntax of flexbox, etc... All these plugins are included in your stylecow installation, so no extra installations are required.

Learn how to create your own plugins

The available plugins are the following:

Support for CSS incoming features:

color: manipulate colors using the CSS Colors Module Level 4 syntax
matches: add support for :matches() selector
calc: add support for calc() function (only if no more than one unit is used)
custom-media: add support for @custom-media at-rule
custom-selector: add support for @custom-selector at-rule
variables: allows the use of CSS variables

Language extensions:

nested-rules: unwraps nested rules
extend: use placeholders and @extend

Utilities:

import: To insert the code of the @import rules with relative urls

Code fixing:

fixes: Fixes automatically some well-known browsers issues
flex: Generate the css fallback for browsers supporting old syntax of flexbox (webkit and explorer).
webkit-gradient: Generate the css fallback for webkit browsers supporting the old syntax of css gradient.
prefixes Adds automatically all vendor prefixes needed. Uses the caniuse database to keep the prefixes updated.
rem Adds fallback for rem values in pixels

Emulation of some features in old Internet Explorer using ms-filters:

msfilter-background-alpha Emulate the rgba() background in Internet Explorer 8 and lower using ms filters
msfilter-linear-gradient Emulate the linear-gradient() background in Internet Explorer 8 and lower using ms filters
msfilter-transform Emulate the CSS 2d transformations in Internet Explorer 8 and lower using ms filters

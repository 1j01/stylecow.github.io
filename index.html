Why another CSS preprocessor?

The main purpose of stylecow is to bring more CSS support to all browsers. Stylecow converts your code to be more compatible with all browsers adding vendor prefixes, fixing bugs, generating polyfills, to emulate some non-supporting css properties, etc.

You can also play with new (or future) css features such variables, nested rules or color manipulation using the w3c syntax. If you need more power, you can use other tools like SASS or LESS and use stylecow only to fix your code for more compatibility.

Quick start

Use npm to install the library:

npm install stylecow

You have to install also some plugins. For example, to play with colors and variables

npm install stylecow-plugin-color stylecow-plugin-variables

Now you need to create a config file where to save the minimal browser supported, code style, plugins used, etc. To create it in an interactive way:

stylecow init

An example of config file:

{
	"input": "styles.css",
	"output": "styles.min.css",
	"support": {
		"explorer": 8,
		"firefox": 30,
		"chrome": 35,
		"safari": 5,
		"opera": 22,
		"android": 4,
		"ios": 6
	},
	"plugins": [
		"color",
		"variables"
	],
	"code": "minify"
}

The input and output properties are optional you can define it later using --input and --output options

To execute the config file:

stylecow execute stylecow_config.json

You can use the option --watch to keep stylecow watching the css files and regenerate the output code when any input file changes.

Use stylecow execute -h to view all available options.

Programatic usage

Here's an example of using stylecow in node:

var stylecow = require('stylecow');

stylecow.setConfig({
	support: {
		explorer: 8,
		firefox: 30,
		chrome: 35,
		safari: 5,
		opera: 22,
		android: 4,
		ios: 6
	},
	plugins: [
		"color",
		"variables"
	],
	code: "minify"
});


css = stylecow.convertFromFile("styles.css");

var code = css.toCode();

Plugins

All css transformations are done via plugins. There are several plugins available, for example, to play with variables using the CSS Variables Module 1 syntax, to manipulate the colors using the CSS Colors Module Level 4, to work with Flexible Box Layout Module Level 1 generating the old syntax for ie10 and webkit browsers, etc...

The available plugins are the following:

Support for CSS incoming features:

  • color
  • matches
  • nested-rules
  • variables

Utilities:

  • import (To insert the code of the @import rules with relative urls)
  • source-comment

Code fixing:

  • fixes
  • flex
  • initial
  • linear-gradient
  • prefixes
  • rem

Emulation of some features in old Internet Explorer using ms-filters:

  • msfilter-background-alpha
  • msfilter-linear-gradient
  • msfilter-transform

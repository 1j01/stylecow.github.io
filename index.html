Why another CSS preprocessor?

The main purpose of stylecow is to bring more CSS support to all browsers. Stylecow converts your code to be more compatible with all browsers adding vendor prefixes, fixing issues, generating polyfills, etc.

You can play also with latests (or futures) css features such variables, nested rules or color manipulation using the w3c syntax. This mean you don't have to learn another language such LESS or SASS to use variables or color manipulation, you write css and you get css, and if you stop using stylecow, your css code remains css code.

Quick start

Use npm to install the library and the plugins to use. The best way is creating a package.json file with the modules name:

{
  "dependencies": {
    "stylecow": "*",
    "stylecow-plugin-color": "*",
    "stylecow-plugin-variables": "*",
    "stylecow-plugin-nested-rules": "*"
  }
}

And then use npm to install:

npm install

Now you have to create the javascript file to execute it:

var stylecow = require('stylecow');

stylecow.setConfig({

    //Minimal browser supported
    support: {
        explorer: 8,
        firefox: 30,
        chrome: 35,
        safari: 5,
        opera: 22,
        android: 4,
        ios: 6
    },

    //Plugins to execute (without "stylecow-plugin-")
    plugins: [
        "color",
        "variables",
        "nested-rules"
    ],

    //Code style used (normal or minify)
    code: "minify"
});

//Load and convert a css file
css = stylecow.convertFromFile("styles.css");

//Get the minified code
var code = css.toCode();

Command line interface

A better way to execute stylecow is using the command line interface. To do that, you need to install the stylecow-cli module. Use -g for global installation:

npm install stylecow-cli -g

Now there is the stylecow command available with some useful utilities. As you need a json file with the configuration, you can create it in an interactive way:

stylecow init

A typical config file looks like:

{
    "input": "styles.css",
    "output": "styles.min.css",
    "support": {
        "explorer": 8,
        "firefox": 30,
        "chrome": 35,
        "safari": 5,
        "opera": 22,
        "android": 4,
        "ios": 6
    },
    "plugins": [
        "color",
        "variables",
        "nested-rules"
    ],
    "code": "minify"
}

The input and output properties are optional you can define them later using --input and --output options

To execute the config file:

stylecow execute stylecow.json

You can use simply stylecow execute if your config file is "stylecow.json" (the default name).

The option --watch keeps stylecow watching the css files to regenerate the output code when changes are detected.

If you want your browser reloads the css automatically, use the option --socket to create a websocket server that comunicates with your browser and reloads the css when changes are detected (you don't need use --watch with --socket because it's done automatically). When a websocket is created, some javascript code is needed to place in the html, but you can create a bookmarklet to get always available, like this:
stylecow live reload

Use stylecow execute -h to view all available options.

Using with Grunt

Are you one of these guys? No problem, there is a grunt plugin available that you can install with npm:

npm install grunt-stylecow --save-dev

Now, you can register and configurate it in the Grunt file:

module.exports = function(grunt) {
    grunt.initConfig({
        stylecow: {
            options: {
                "support": {
                    "explorer": 10,
                    "firefox": 30,
                    "chrome": 35,
                    "safari": 6,
                    "opera": 22,
                    "android": 4,
                    "ios": 6
                },
                "plugins": [
                    "color",
                    "variables",
                    "nested-rules"
                ],
                "code": "normal"
            },
            dist: {
                files: {
                    'css/styles.min.css': ['./css/styles.css']
                }
            }
        }
    });

    grunt.loadNpmTasks('grunt-stylecow');

    grunt.registerTask('default', ['stylecow']);
}

Or if there is a config file that you want to reuse:

module.exports = function(grunt) {
    grunt.initConfig({
        stylecow: {
            options: require('./stylecow.json'),
            dist: {
                files: {
                    'css/styles.min.css': ['./css/styles.css']
                }
            }
        }
    });

    grunt.loadNpmTasks('grunt-stylecow');

    grunt.registerTask('default', ['stylecow']);
}

Plugins

All stylecow operations are done via plugins. There are several plugins available, for example, to play with variables using the w3c syntax, to manipulate the colors using the CSS Colors Module Level 4 syntax, to play with the latest syntax of flexbox generating the old syntax for ie10 and webkit browsers, etc...

The available plugins are the following:

Support for CSS incoming features:

color: manipulate colors using the CSS Colors Module Level 4 syntax
matches: add support for :matches() selector
nested-rules: allows nested rules
variables: allows the use of CSS variables

Utilities:

import: To insert the code of the @import rules with relative urls
source-comment: Adds comments like /* css/styles.css:12 */ before with the location of each rule in the original code

Code fixing:

fixes: Fixes automatically some well-know browsers issues
flex: To use the latest syntax of flexbox, generating the fallback for browsers supporting deprecated syntax.
initial: Add support for initial value
prefixes Adds automatically all vendor prefixes needed
rem Adds fallback for rem values in pixels

Emulation of some features in old Internet Explorer using ms-filters:

msfilter-background-alpha Emulate the rgba() background in Internet Explorer 8 and lower using ms filters
msfilter-linear-gradient Emulate the linear-gradient() background in Internet Explorer 8 and lower using ms filters
msfilter-transform Emulate the CSS 2d transformations in Internet Explorer 8 and lower using ms filters

Why another CSS preprocessor?

The main purpose of stylecow is to bring more CSS support to all browsers. Stylecow converts your code to be more compatible with all browsers adding vendor prefixes, fixing bugs, generating polyfills, to emulate some non-supporting css properties, etc.

You can also play with new (or future) css features such variables, nested rules or color manipulation using the w3c syntax. If you need more power, you can use other tools like SASS or LESS and use stylecow only to fix your code for more compatibility.

Quick start

Use npm to install the library (you can use -g to install it globally):

npm install stylecow -g

First you have to create a config file where to save the minimal browser supported, code style, plugins used, etc. To create it in an interactive way:

stylecow config --create

Now you have a config file like like the following:

{
	"input": "styles.css",
	"output": "styles.min.css",
	"support": {
		"explorer": 8,
		"firefox": 30,
		"chrome": 35,
		"safari": 5,
		"opera": 22,
		"android": 4,
		"ios": 6
	},
	"plugins": [
		"color",
		"flex",
		"variables"
	],
	"code": "minify"
}

As you can see, the input file "styles.css" is loaded and transformed according with the minimal browser support defined in "support" property, executing the plugins "color", "flex" and "variables". The output code will be minified and saved in styles.min.css.

To execute the config file:

stylecow execute my-config.json

You can use the option --watch to keep stylecow watching the css files and regenerate the output code when any input file changes.

Use stylecow execute -h to view all available options.

Programatic usage

Here's an example of using stylecow in node:

var stylecow = require('stylecow');
var css = stylecow.readFile('styles.css');
var codeStyle = stylecow.getCodeStyle('minify');

css.convert([
	"color",
	"flex",
	"variables"
], {
	"explorer": 8,
	"firefox": 30,
	"chrome": 35,
	"safari": 5,
	"opera": 22,
	"android": 4,
	"ios": 6
});

var code = css.toCode(codeStyle);

Plugins

All css transformations are done via plugins. Each plugin deals with a specific css feature. For example, there is a plugin to play with variables using the CSS Variables Module 1 syntax, other to manipulate the colors using the CSS Colors Module Level 4 and other to work with Flexible Box Layout Module Level 1 that generates the fallback code for browsers supporting the old syntax (ie10 and some webkit browsers).

The available plugins are the following:

Support for CSS incoming features:

  • color
  • matches
  • nested-rules
  • variables

Utilities:

  • import (To insert the code of the @import rules with relative urls)
  • source-comment

Vendor prefixes, polyfills, browser bugs fixers:

  • animation
  • appearance
  • background
  • border
  • box-shadow
  • box-sizing
  • calc
  • clip
  • column
  • cursor
  • document
  • flex
  • float
  • fullscreen
  • grid
  • initial
  • inline-block
  • linear-gradient
  • mask
  • min-height
  • object
  • opacity
  • pseudoelements
  • transform
  • transition
  • type
  • region
  • rem
  • sizing
  • sticky
  • user-select
  • vmin

Emulation of some features in old explorers using ms-filters:

  • msfilter-background-alpha
  • msfilter-linear-gradient
  • msfilter-transform

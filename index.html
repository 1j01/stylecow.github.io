Why another CSS preprocessor?

The main purpose of stylecow is support modern CSS in all browsers. Stylecow transform your code to be compatible with all browsers adding vendor prefixes, fixing issues, generating polyfills, etc.

You can play also with latests (or futures) css features such variables, nested rules or color manipulation using the w3c syntax. This mean you don't have to learn another language such LESS or SASS to use variables or color manipulation, you write css and you get css, and if you stop using stylecow, your css code remains css code.

Quick start

Use npm to install the library. Add -g if you want install it globally:

npm install stylecow -g

Now you have the stylecow command with some useful utilities. Run stylecow -h to display the help for usage. The next step is create a json file with the configuration of the css conversion. You can create it in an interactive way using the command:

npm stylecow init

In this process, you have to define the following settings:

  • support: Minimum version supported by the browsers explorer, firefox, chrome, safari, opera, android and ios
  • plugins: The plugins you want to use in the conversion (see below)
  • code: Style used to ouput the css code. For example: normal and minify
  • input: path of the css file to convert
  • output: path of the file where save the result
  • map: whether or not generate the source map

A typical config file looks like:

{
    "files": [
        {
            "input": "styles.css",
            "output": "styles.min.css",
            "map": "styles.min.map"
        }
    ],
    "support": {
        "explorer": 8,
        "firefox": 30,
        "chrome": 35,
        "safari": 5,
        "opera": 22,
        "android": 4,
        "ios": 6
    },
    "plugins": [
        "color",
        "variables",
        "nested-rules"
    ],
    "code": "minify"
}

By default the configuration file is saved as stylecow.json but you can choose a different name. Now, to convert your css just execute:

stylecow

You can use the following options:

  • -c, --config <file> Change the default config file (./stylecow.json)
  • -w, --watch To watch the changes in the input file and generate the output automatically
  • -r, --live-reload Live reload the css in the browser when the output changes (using a websocket)

Using with Grunt/Gulp

Are you one of these guys? No problem, there is a grunt plugin available that you can install with npm:

npm install grunt-stylecow --save-dev

Now, you can register and configurate it in the Grunt file:

module.exports = function(grunt) {
    grunt.initConfig({
        stylecow: {
            options: require('./stylecow.json'), //Your config file
            dist: {
                files: {
                    'css/styles.min.css': ['./css/styles.css']
                }
            }
        }
    });

    grunt.loadNpmTasks('grunt-stylecow');

    grunt.registerTask('default', ['stylecow']);
}

Or if you are a Gulp-guy, here is your plugin: 

var gulp = require('gulp');
var stylecow = require('gulp-stylecow');

gulp.task('stylecow', function() {
  gulp.src('./css/styles.css')
    .pipe(stylecow(require('./stylecow.json')))
    .pipe(gulp.dest('./dist/'))
});

Plugins

All stylecow operations are done via plugins. There are several plugins available, for example, to play with variables using the w3c syntax, to manipulate the colors using the CSS Colors Module Level 4 syntax, to play with the latest syntax of flexbox generating the old syntax for ie10 and webkit browsers, etc... All these plugins are included in your stylecow installation, so no extra installations are required.

The available plugins are the following:

Support for CSS incoming features:

color: manipulate colors using the CSS Colors Module Level 4 syntax
matches: add support for :matches() selector
nested-rules: allows nested rules
variables: allows the use of CSS variables

Utilities:

import: To insert the code of the @import rules with relative urls

Code fixing:

fixes: Fixes automatically some well-know browsers issues
flex: To use the latest syntax of flexbox, generating the fallback for browsers supporting deprecated syntax.
prefixes Adds automatically all vendor prefixes needed
rem Adds fallback for rem values in pixels

Emulation of some features in old Internet Explorer using ms-filters:

msfilter-background-alpha Emulate the rgba() background in Internet Explorer 8 and lower using ms filters
msfilter-linear-gradient Emulate the linear-gradient() background in Internet Explorer 8 and lower using ms filters
msfilter-transform Emulate the CSS 2d transformations in Internet Explorer 8 and lower using ms filters
